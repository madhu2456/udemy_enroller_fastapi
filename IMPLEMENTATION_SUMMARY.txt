QUICK MONITORING COMMANDS
================================================================================

Show All Bulk Checkout Metrics:
  grep "📊" logfile.txt

Show All Batch Summaries:
  grep "📦" logfile.txt

Show Both:
  grep -E "(📊|📦)" logfile.txt

Find Sessions That Got Blocked:
  grep "Session_Blocks=[1-9]" logfile.txt

Find Batches With Many 403 Recoveries:
  grep "403_Recoveries=[3-9]" logfile.txt

Find Failed Batches (0% success):
  grep "Success_Rate=0%" logfile.txt

Real-Time Docker Logs:
  docker logs -f <container_name> 2>&1 | grep "📊"

Calculate Average Success Rate:
  grep "Success_Rate=" logfile.txt | grep -oP 'Success_Rate=\K[0-9.]+' | \
    awk '{sum+=$1; count++} END {print "Average:", sum/count "%"}'

Count Total 403 Recoveries:
  grep "403_Recoveries=" logfile.txt | grep -oP '403_Recoveries=\K[0-9]+' | \
    awk '{sum+=$1} END {print "Total:", sum}'

METRIC INTERPRETATION GUIDE
================================================================================

Attempts (Checkout Tries)
  Good:    1-2 (no retries needed)
  Warning: 3-5 (some issues)
  Bad:     >5  (major problems)

403_Recoveries (CSRF Refreshes)
  Good:    0-1 per batch
  Warning: 2-3 per batch (rate limiting)
  Bad:     >3 per batch (session unstable)

Session_Blocks (IP Detected)
  Good:    0 (never blocked)
  Bad:     >0 (IP detected as automated)

Success_Rate (% Enrolled)
  Good:    >90%
  Warning: 70-90% (degraded)
  Bad:     <50% (major issues)

Duration (Batch Processing Time)
  Good:    <2 minutes per batch
  Warning: 2-3 minutes per batch
  Bad:     >3 minutes per batch

Total_Delay (Time Spent Waiting)
  Good:    <10 seconds
  Warning: 10-20 seconds
  Bad:     >20 seconds

DECISION TREE: WHEN TO ADD FIRECRAWL API KEY
================================================================================

Add API key when you see ANY of:
  ✗ Session_Blocks > 0 in any batch
  ✗ Success_Rate < 70% for 3 consecutive batches
  ✗ 403_Recoveries > 2 in a single batch
  ✗ Multiple "Too many 403 errors" messages

Keep free solution when you see:
  ✅ Success_Rate > 85% consistently
  ✅ 403_Recoveries < 1 per batch
  ✅ Session_Blocks = 0 always
  ✅ No "Too many 403 errors" messages
