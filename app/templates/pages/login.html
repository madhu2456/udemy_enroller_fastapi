Udemy Enroller — FastAPI Automation Case Study

A private FastAPI and Playwright workflow-orchestration case study demonstrating async queues, bounded concurrency, session-state handling, telemetry logging, and background processing.

View Architecture Demonstrating high-throughput background processing
View source code and architecture on GitHub

Async Workflows

Utilizes FastAPI background tasks and asyncio to orchestrate complex external integrations.

Session Management

Demonstrates secure handling of session state and bounded concurrency for API rate limits.

Data Pipeline

Implements dynamic filtering and processing logic for robust data pipelines.

System Telemetry

Real-time logging via SSE showcasing isolated per-user telemetry streams.

Architecture Overview

High-throughput orchestration components powering the case study.

FastAPI Gateway

Asynchronous API endpoints handling HTTP requests and SSE telemetry streams.

Celery Worker Queue

Distributes long-running workflow processes across multiple horizontal worker nodes.

Redis Broker

High-throughput message broker coordinating sub-tasks between FastAPI and Celery.

PostgreSQL Records

Persistent relational storage for application state, run telemetry, and logs.

Playwright Runner

Headless browser automation engine executing dynamic, obfuscated workflows.

SSE Telemetry

Real-time Server-Sent Events propagating asynchronous task status to the frontend.

Demo Mode

This public demo uses mocked workflow events and does not collect credentials, passwords, cookies, tokens, or third-party account data.
