Architecture & Workflow Guides

Technical guides for the production architecture behind Udemy Enroller.

Beginner Friendly

System Architecture Overview

Udemy Enroller uses FastAPI to coordinate asynchronous enrollment workflow runs, Celery and Redis for background processing, PostgreSQL for durable workflow records, SQLAlchemy for transactional persistence, Playwright for browser-workflow orchestration, and SSE for real-time telemetry.

Session-State Handling

Udemy Enroller is designed around secure, user-authorized session-state handling. Sensitive authentication data should be protected through encryption, access controls, environment isolation, and short-lived credentials where possible.

Data Pipeline & Telemetry

The platform validates workflow inputs, applies filtering rules, records task outcomes, and streams isolated execution logs through Server-Sent Events.

Self-Hosting with Docker

The project is containerized for deployment on a modern VPS using Docker.

cd udemy_enroller_fastapi
